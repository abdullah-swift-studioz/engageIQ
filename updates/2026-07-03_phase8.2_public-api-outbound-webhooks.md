## What Was Built

A versioned public REST API (`/api/v1/public/*`) authenticated by EngageIQ API keys with per-key **scopes**, plus a full **outbound-webhook** system (merchant-configurable endpoints, HMAC-signed, encrypted secret at rest, delivery worker with retries + a durable delivery log), plus a monochrome **Settings UI** to manage both. All against the FROZEN schema (`ApiKey.scopes`, `OutboundWebhook`, `WebhookDelivery` from freeze-v2) — no schema or migration changes.

### Public REST API — `/api/v1/public/*`
Auth: `Authorization: Bearer eiq_...` → `publicApiKeyAuth` (prefix lookup + bcrypt compare + expiry check + `lastUsedAt` stamp), then a per-route `requireScope(...)` preHandler. Every query is merchant-scoped. Rate-limited per API key (600/min; events 1000/min).

| Route | Scope | Notes |
|---|---|---|
| `GET /customers` (+`?search`,`?page`) | `customers:read` | safe profile projection |
| `GET /customers/:id` | `customers:read` | |

### Outbound webhooks
- **Config CRUD** (JWT, gated on `api_keys:manage`): create (secret shown once), list (secret hidden), get (+recent deliveries), patch, delete, **rotate-secret**, **send test ping**, deliveries log.
- **Secret at rest:** AES-256-GCM (`WEBHOOK_ENCRYPTION_KEY`, base64/hex 32 bytes; dev-derives from `JWT_SECRET` with a warning). Signing: HMAC-SHA256 → `X-EngageIQ-Signature: sha256=<hex>` over the raw body (+ `X-EngageIQ-Event`, `X-EngageIQ-Delivery` headers).
- **Delivery worker** (`webhook-delivery` queue, 6 attempts, exp backoff 5s): POSTs with timeout (`WEBHOOK_DELIVERY_TIMEOUT_MS`), updates the `WebhookDelivery` row (statusCode, attempts, success, error, deliveredAt, nextRetryAt), throws to drive BullMQ retry. Drops (Unrecoverable) if the webhook was deleted/deactivated or the secret can't be decrypted.
- **`emitOutboundEvent(merchantId, event, payload)`** — fire-and-forget, tenant-scoped, NEVER throws into callers. Matches active webhooks subscribed to the event, writes a delivery row, enqueues a job.

## Decisions Made This Session
- **Additive emit hooks now** (user-approved) rather than infra-only — delivers a working end-to-end feature. Each hook is a one-line, tagged, fire-and-forget call; `emitOutboundEvent` swallows all errors so it can never break a segment/campaign/scoring operation.
- **AES-256-GCM for the secret at rest**, versioned (`v1:iv:tag:ct`), with a dev key derived from `JWT_SECRET` (warned) so the app boots credential-free; required in production.
- **Scopes are a separate taxonomy from RBAC permissions** (`PUBLIC_API_SCOPES`), stored on `ApiKey.scopes`. Settings management is gated on the existing `api_keys:manage` RBAC permission (Owner/Admin/Agency-Admin).
- **Distinct nav entry** instead of claiming `/settings` (owned by the RBAC lane), to avoid a cross-lane collision.
- **Rate-limit keyed by the bearer token** (per-key), independent of auth-hook ordering.
- Added `name: true` to the dynamic-segment select in `segment-evaluator.ts` so the webhook payload can carry `segmentName` (additive, tagged).
